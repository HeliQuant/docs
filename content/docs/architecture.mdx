HeliQuant is structured like a trading firm: specialist **desks** produce domain reads, **researchers**
argue bull-vs-bear, and a **Portfolio Manager** synthesizes one disciplined decision per cycle — and, via
the [PM sign-off merge](/docs/machinery#the-merge-the-floor-proposes-the-llm-pm-disposes), that same PM now
**signs off on every trade the continuous floor proposes**. Seven desks call an LLM; two more (flow-intel
and whale) fold in as pre-computed stances.

## How a decision flows

Every desk pulls **real data** (no placeholders) from a live source, an LLM interprets it into a stance,
the debate stress-tests it, and the PM turns the weighted picture into a single recorded decision.

## The 3-phase loop

<Steps>
<Step>
### Phase 1 — Analysis
Each of the specialist desks runs a **real tool** (no placeholders) over fresh data and returns a stance
+ confidence. Different desks run on different LLM models so they don't groupthink.
</Step>
<Step>
### Phase 2 — Debate
A **bull** and a **bear/risk** researcher argue using only the desk outputs, then rebut each other
(deep model, 2 rounds). Dissent is surfaced, not averaged away.
</Step>
<Step>
### Phase 3 — Decision (PM)
The Portfolio Manager weighs the desks **ranked by learned reliability**, applies a hard
**risk/reward ≥ 2:1 gate**, and outputs `ENTER` (only on a validated edge) or `ABSTAIN`. The decision's
hash is anchored on Mantle.
</Step>
</Steps>

## The discipline gates

- **No validated edge → ABSTAIN.** `validated_edges.json` is allowed to be empty. An edge must be
  *earned* (see [Self-learning](/docs/self-learning)).
- **R:R ≥ 2:1.** Even a validated edge abstains when the structure doesn't pay.
- **Macro event-risk.** Around FOMC/CPI, BTC/ETH **de-risk** — volatility ~2× but direction is noise
  (research-backed), so the firm won't bet direction into a scheduled print.
- **AGGRESSIVE is earned.** Fractional-Kelly, capped at 3% risk / 5× leverage, with a 20% drawdown
  breaker — unlocked **only** when an OOS-validated edge's live signal fires.

## The continuous trading floor

Alongside the LLM org, a **campaign** runs a continuous paper floor at live prices. It computes its **own
deterministic desk votes** — flow-intel signals, z-score positioning extremes, and the Hyperliquid whale
read, with **no LLM call** (`campaign.py · desk_votes`) — *and then*, as the final gate, asks the **LLM PM
to sign off** on the candidate (the [merge](/docs/machinery#the-merge-the-floor-proposes-the-llm-pm-disposes)).
Before that sign-off, a setup must clear an ordered [**entry-quality pipeline**](#the-entry-quality-pipeline)
(below). Approved positions are sized on a **warm/cold ladder** (proven winners boosted up to 1.6×, proven
losers half-size + throttled) and timed on a **dynamic horizon** (3h fade / 4h trend / 24h validated edge).
Stops are **calibrated per asset** from realized excursions (MFE/MAE — the fixed 2.5× TP was reached only
0–21% of the time), trend trades **trail** to let a run go, and it learns from every close **and every
abstention**. Every position is a little state machine:

### The entry-quality pipeline

A candidate setup must clear an **ordered stack of discipline gates** before the floor opens it. Each gate
is a *per-asset* skip — abstaining one asset never stalls the rest (the loop scans the whole basket every
step). The goal is not to **predict** tops (impossible on an efficient tape) but to **systematically refuse
bad entries**. The live, deployed gate list is surfaced on `/campaign` as `entry_gates`.

The three green gates were added **2026-06-14** to stop the floor buying exhausted moves — the recurring
"entered at the top, TP never reached" failure:

- **TV 1h-vs-1D alignment** — reads TradingView's free multi-timeframe consensus. When the **daily** trend
  opposes the 1h signal, the entry is a counter-daily bounce; the floor fades it (half size, short hold) or,
  on strong opposition, abstains.
- **Smart-money whale veto** — when **high-conviction** Hyperliquid whales (≥2, real net notional, proven
  ROE) sit *against* the trade, the floor won't buy the top they're shorting. The LLM org already reasons
  about this whale-vs-price divergence; this is its algorithmic counterpart on the no-LLM floor.
- **Over-extension** — refuses a fresh entry once price has already run **>2.5 ATR past its 1h mean** (or is
  decelerating). Enter a trend *early / on a pullback*, never the stretched tail.

The **final** gate is the **LLM PM sign-off** (the
[merge](/docs/machinery#the-merge-the-floor-proposes-the-llm-pm-disposes)): once a candidate clears every
algorithmic gate, the floor hands the **live signals** to the LLM Portfolio Manager for a one-call
APPROVE / VETO. It **fails open** on an LLM outage so the floor never freezes — the algorithmic gates have
already vetted the setup.

A symmetric **fee-breakeven guard** sits on the exit side: a near-TP lock only banks a win that clears the
round-trip cost, so a sub-fee "win" (`NEARTP`) is never booked. None of these is an **edge** — they are
**discipline**. The floor's honest expectation on a no-edge directional book is still ~breakeven; the only
validated positive edge remains delta-neutral funding-carry (HYPE **+5.8%/yr** walk-forward). Rigorous,
un-flattering metrics on the real ledger (Sharpe, Sortino, max-drawdown, with annualised figures *withheld*
until the sample earns them) are served at `/performance` and shown on the **Dyno** page.

The floor is **self-driving**: any read of `/campaign` or `/trades` (and every keep-alive ping of
`/run-cycle`) opportunistically advances a step in the background — lock-guarded, **opening new positions
at most once every `CAMPAIGN_STEP_MIN` = 4 min** while **resolving matured positions every ~60 s** so exits
stay snappy — and it keeps trading on real traffic even if the external pinger lapses.

## Stack

- **Brain:** multi-provider LLM abstraction (BYO keys) · runs clean on free Groq
- **Data:** Bybit (market) · Hyperliquid (whale) · Allora (decentralised-AI macro) · DeFiLlama (Mantle
  fundamentals) · Nansen / Elfa (smart-money & social) · Etherscan v2 (on-chain)
- **Chain:** Mantle Sepolia — ERC-8004 (identity/reputation/validation) · ERC-8183 (JobManager) ·
  ERC-7857 (transferable agent NFT) · TradingVault · AlloraConsumer
- **Always-on:** Railway service + Supabase state · driven by an uptime pinger + self-driving floor
