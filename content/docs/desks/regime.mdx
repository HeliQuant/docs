**ERC-8004 tokenId 3.** This is **the ML desk** — and it's where HeliQuant's honesty about machine
learning lives. It runs the model that *earned* its place (a regime classifier) and explicitly does
**not** run the one that didn't (a directional predictor).

## In / Process / Out

`firm/organization.py · tool_regime()` (line 93) over each asset's `data/{ticker}_features.csv`:

- **In:** the latest feature row (ADX, `ema_slope_5`, `volatility_10`) + ADX/vol quantile thresholds.
- **Process:** `scripts/multi_asset.detect_current` (line 305) maps the row to a regime with **three
  deterministic rules** — high `volatility_10` → `High_Volatility`; else high `adx` → `Trending_Up` /
  `Trending_Down` (by `ema_slope_5` sign); else `Ranging`. A dynamic R:R (2.0–3.0) is scaled from ADX
  strength.
- **Out:** `{current_regime, adx, trending_threshold, trend_following_applicable_now,
  recommended_dynamic_rr, regime_classifier_oos_accuracy, oos_validated_trading_edge, status}`.

### The live label is a rule-based proxy — the XGBoost is the *backtest* engine

This is the honest nuance. The **live** regime label that feeds the PM each cycle is the **rule-based
`detect_current` proxy** above — *not* a fresh XGBoost inference. The **XGBoost classifier** is trained and
scored in the **backtest replay path** (`scripts/multi_asset.py` — `model.predict`, line 260;
`replay()`, line 313); it is not called on the live tick.

The headline **`82.6%` OOS (`91.6%` when confident)** is a **backtested, MNT-scoped** result
(`firm/asset_configs.py:16-19`) — measured on MNT's forward-4h regime, beating persistence (77.5%). The
code emits that **same string for every asset** as a statement of the engine's pedigree; treat it as a
property of the *method*, not a per-asset live accuracy. And — as everywhere on this site — **accurate
regime ≠ trading profit**: `oos_validated_trading_edge` is `false` for every asset today.

## The honest ML story

We tested a **directional** ML (XGBoost predicting BUY/SELL on MNT). It scored **~51% accuracy and
−50% to −62% OOS net of fees** (Finding #22) — no better than a coin flip after costs. It is **not used
and not registered on-chain.** A full-history (2.9 yr) walk-forward + rolling re-tune of trend-following
was also flat-to-negative OOS; the earlier "+5.58%" was a short-window artifact, since scrubbed.

What *is* accurate is the **regime classifier** (82.6% OOS on MNT in backtest). But an accurate regime
label is **not** a trading profit — so it's used as a **capital allocator / risk read** feeding the PM,
never as a price oracle, and the live label is the cheap, responsive `detect_current` proxy rather than a
per-tick model inference. The good model earns its place; the disproven one stays out. That distinction is
the differentiator.
