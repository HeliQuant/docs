**ERC-8004 tokenId 13** (firm child, minted 2026-06-21) — promoted into the PM synthesis 2026-06-20
(previously a supporting input). Reads **Mantle chain fundamentals** from DeFiLlama (free, keyless) and turns them
into an **ecosystem risk-on / risk-off** tilt — chain-level context, not an asset-specific trade signal.

## What it reads

`firm/organization.py · tool_defillama()` (over `firm/defillama_client.py`):

- **Chain TVL** — Mantle total-value-locked + 7d / 30d change + trend (rising / falling / stable).
- **Fees** — 24h chain fees.
- **Staking yields** — top Mantle staking-yield pools.

It maps the TVL trend to `ecosystem_risk`: rising TVL → **risk-ON (capital inflow)**, falling →
**risk-OFF (outflow)**, flat → stable.

## Output (shape — live values fetched each cycle)

```json
{
  "asset": "MNT",
  "mantle_tvl_usd": 0, "tvl_chg7d_pct": 0.0, "tvl_chg30d_pct": 0.0, "tvl_trend": "stable",
  "fees_24h_usd": 0, "top_staking_yields": [],
  "ecosystem_risk": "stable",
  "caveat": "advisory ecosystem context (chain-level), not an asset-specific trade signal"
}
```

## Honest caveats

This is **advisory context**, not an edge: chain TVL / fees describe ecosystem health, not where MNT goes
next. It tilts risk appetite — size up when Mantle capital is flowing in, down when it is leaving — and it
is strongest for MNT and Mantle-native assets.
